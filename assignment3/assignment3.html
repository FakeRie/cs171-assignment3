Assignment 3 - Ray-based Rendering and Loop Subdivision
 

Introduction

In this assignment, you are required to complete two tasks to form your entire assignment. Your first task is to create a rendering program using ray-based direct lighting method. After that, you need to implement a Loop subdivision algorithm to refine the coarse triangle meshes in order to get a better visual result. Ray-based rendering method is the fundamental method when it comes to physically-based and photorealistic rendering. Loop subdivision is a rather basic subdivision scheme that can give you a rough understanding of ways to smooth triangle meshes.

In the following, we will give you the specifics about what you need to accomplish, as well as some related guidelines in order to assist your programming.

Things to be done

- [must] A ray-based rendering program which requires you to implement 5 parts:
--- (1) Generating camera rays;
--- (2) Ray intersection test with a single triangle;
--- (3) Building a uniform grid for triangle meshes as the acceleration structure for ray-tracing;
--- (4) Ray intersection test with triangle meshes using uniform grid;
--- (5) Phong reflection model.
- [must] A subdivision algorithm that smoothes meshes, which requires you to:
--- (1) Subdivide closed surface;
--- (2) Subdivide non-closed surface;
- [optional] A subdivision algorithm with reduced complexities:
--- (1) Reduce the time complexity to o(p^2+f) (lower than c*(p^2+f); c is a constant; "p" is the number of vertices of the object before you do next subdivision and "f" is the number of faces);
--- (2) Reduce the space complexity to O(p^2+f) (no more than c*(p^2+f); c is a constant; "p" is the number of vertices of the object before you do next subdivision and "f" is the number of faces);

(3) Building a uniform grid for triangle meshes
This part should be implemented in <buildUniformGrid> function of <triangleMesh.hpp> file. In this part, you need to:
- Decide the dimensions of the grid for each mesh object;
- Build a data structure for your uniform gird;
- Loop over each triangle to add it into appropriate grid cells.

Reference: Introduction to Acceleration Structures (Grid)

Note that setting the dimensions to be fixed values may however suffer from some minor performance issues.

For the uniform grid, you can design your own data structure that fits your need. It is essentially a 2D array, whose first dimension is the total size of the grid cells and the second dimension is the maximum number of triangles in one grid cell. Making it an array of dynamic length will save you some effort determining the second dimension. A useful tip is that you can always convert a 3D coordinate to a 1D integer, so you only need one dimension to store the size of grid cells. However, if you wish to use a 4D array for that, it's no difference really.

To determine whether a triangle is contained by a grid cell, you can first determine each triangle's bounding box, and add triangle to the grids that overlap with it:

Reference: Introduction to Acceleration Structures (Grid)

If you want, you could have a more careful test to see whether triangle itself intersectes with the grid, not the triangle's bounding box. This process saves both traversal time and space, nevertheless, it is not of necessity here. So there is no further explaination about the triangle-box intersection. If you are interested, please refer to Akenine-Mollser T. Fast 3D triangle-box overlap testing.

(4) Ray intersection test with triangle meshes using uniform grid
The algorithm should be implemented in <rayIntersection> function of <triangleMesh.hpp> file. The interface of the function is: 


    bool rayIntersection(Interaction& interaction, const Ray& ray);
Interaction class, as always, is the place where you set the intersection information when it is true for intersection test. However, notice that before every test with a particular shape, we will first test with the shape's bounding box and we only go into shape intersection test when it's true for intersection test with the shape's bounding box. In this case, entryDist and exitDist store the t values that the ray enters and exits the bounding box, which indicates the points the ray enters and leaves the uniform grid. This information should be helpful to you.

The essential idea here is to conduct an efficient grid traversal algorithm that can traverse the grid in correct order so that we can return the result when we find the first correct intersection, making the rest of the intersection testes unnecessary to be conducted. Grid cell traversal algorithm is also a quite complete algorithm, which is described in detail here: Amanatides J, Woo A. A fast voxel traversal algorithm for ray tracing. Since the paper is both clear and concise and the algorithm is rather simple, we are not going into the details of this algorithm further. Instead, here we provide some useful tips for implementation.

The first one is to keep track of numerical errors, such as divison by 0. The second one is that, in the algorithm we mentioned above, when we are talking in units of t, whether it's tMax or tDelta, they are always positive value, because a ray does not go backwards. The last one is that when you are calculating the grid's coordinate, be careful that it might go out of the boundary. Always conduct appropriate boundary treatment to avoid array's index out of bounds or incorrect results.

(5) Phong reflection model
This part should be implemented in <radiance> function of <directLightingIntegrator.hpp> file. The interface of the function is: 


    Eigen::Vector3f radiance(Interaction* interaction, Ray * ray);
Here, Interaction class is not where you put information, but where you read information. It includes all the intersection information you need. Variable of Ray class is the ray intersecting the shape. We need to return a Eigen::Vector3f containing the color of this point.

Before we calculate the radiance of the intersection point, we need to first decide whether the point is in shadow. Since we are only considering direct lighting, the process is rather simple. All we need to do is to shoot another ray from the intersection point to the light source. If it intersects with any object, then we can consider that the point is in shadow. In this case we only need to apply ambient color and a fraction of its own color to it. The process is illustrated here:

If it is not in shadow, we need to calculate color using Phong or Blinn-Phong reflection model. This should be quite simple for you now, if you forget something about this, we refer you to Phong reflection model - Wikipedia or Blinn-Phong reflection model - Wikipedia. Just remember not to implement these in GLSL, but in C++ (lol).


B. Loop Subdivision:
Loop Subdivision (named after Charles Loop) is a standard subdivision scheme for triangle meshes. This part needs to be implemented in <subdivision> function of <triangleMesh.hpp> file, <build_data_struct> function and <loop_subdivision> function of <refine.hpp> file. At a high level, it consists of two basic steps:
- 1.Split each triangle into four by connecting edge midpoints (sometimes called "4-1 subdivision").
- 2.Update vertex positions as a particular weighted average of neighboring positions.
